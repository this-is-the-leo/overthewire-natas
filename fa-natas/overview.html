ناتاس چیست؟

ناتاس یک بازی جنگی (Wargame) در حوزه امنیت وب از OverTheWire است که مفاهیم بنیادین امنیت برنامه‌های وب را از طریق یک سری چالش‌های مرحله‌ای آموزش می‌دهد.

// درباره بازی

ناتاس یک بازی جنگی امنیت وب است که برای آموزش مفاهیم بنیادین امنیت برنامه‌های وب طراحی شده است. هر مرحله یک برنامه وب آسیب‌پذیر با یک نقص امنیتی خاص را ارائه می‌دهد. هدف این است که آسیب‌پذیری را پیدا کرده و آن را بهره‌برداری کنید تا رمز عبور مرحله بعد را به دست آورید.

چالش‌ها با مفاهیم پایه مانند مشاهده کد منبع HTML شروع می‌شوند و به تدریج به موضوعات پیشرفته‌تری مانند تزریق SQL، تزریق دستورات، تزریق بین‌سایتی (XSS)، گنجاندن فایل، آسیب‌پذیری‌های سریال‌زدایی و حملات XML External Entity (XXE) می‌رسند.

برخلاف بسیاری از پلتفرم‌های CTF، ناتاس به گونه‌ای طراحی شده است که برای مبتدیان قابل دسترس باشد و در عین حال تجارب یادگیری ارزشمندی برای توسعه‌دهندگان با تجربه فراهم کند. هر مرحله بر دانش قبلی بنا می‌شود و یک مسیر یادگیری ساختاریافته در چشم‌انداز آسیب‌پذیری‌های امنیت وب ایجاد می‌کند.

// چرا این راهنما وجود دارد؟

این راهنما برای ارائه یک منبع یادگیری جامع برای هر کسی که در حال گذراندن بازی ناتاس است، ایجاد شده است. در حالی که حل چالش‌ها لذت‌بخش است، درک آسیب‌پذیری‌های اساسی و یادگیری نحوه پیشگیری از آنها ارزش واقعی این تمرین است.

هر مرحله در این راهنما شامل موارد زیر است:

  • ▸ یک توضیح دقیق از آسیب‌پذیری
  • ▸ دستورالعمل‌های راه حل گام‌به‌گام با رویکردهای متعدد
  • ▸ تحلیل کد کد منبع آسیب‌پذیر
  • ▸ اقدامات دفاعی عملی برای جلوگیری از آسیب‌پذیری
  • ▸ نکات کلیدی برای تثبیت یادگیری

چه یک مبتدی باشید که برای اولین بار امنیت وب را یاد می‌گیرید و چه یک توسعه‌دهنده با تجربه که به دنبال عمق‌بخشی به درک خود هستید، این راهنما تلاش می‌کند دانش واضح و عملی را ارائه دهد.

// اهداف پروژه

این راهنما بر اساس چهار هدف اصلی ساخته شده است:

📖
توضیحات واضح
هر مرحله با دستورالعمل‌های گام‌به‌گام، تحلیل کد و تفکیک مفهومی توضیح داده شده است تا موضوعات پیچیده را قابل درک کند.
🔍
بررسی عمیق آسیب‌پذیری‌ها
درک کنید چرا هر آسیب‌پذیری وجود دارد و چگونه می‌توان از آن بهره‌برداری کرد. یاد بگیرید مانند یک مهاجم فکر کنید.
🛡️
اقدامات دفاعی
با توصیه‌های امنیتی عملی، روش‌های کدنویسی امن و استراتژی‌های دفاع عمیق، نحوه پیشگیری از هر آسیب‌پذیری را بیاموزید.
💻
مثال‌های عملی
شامل قطعه‌های کد، دستورات curl، اسکریپت‌های پایتون و نمایش‌های عملی برای نشان دادن نحوه عملکرد بهره‌برداری در عمل است.
// چه چیزی یاد خواهید گرفت؟

با کار بر روی این مراحل و دنبال کردن این راهنما، یک درک عملی از رایج‌ترین آسیب‌پذیری‌های امنیت وب را توسعه خواهید داد. شما یاد خواهید گرفت که مسائل زیر را شناسایی، بهره‌برداری و پیشگیری کنید:

🟢 مبانی
از اصول اولیه شروع می‌کنید، یاد می‌گیرید که چگونه کد منبع را مشاهده و تحلیل کنید، محدودیت‌های سمت کلاینت را دور بزنید و فایل‌ها و دایرکتوری‌های پنهان را کشف کنید. این مهارت‌های بنیادین برای هر کار امنیتی ضروری هستند.
🟠 آسیب‌پذیری‌های سمت سرور
به عمق نقص‌های امنیتی سمت سرور بروید:
  • گنجاندن فایل محلی (LFI) — خواندن فایل‌های دلخواه از طریق گنجاندن ناامن فایل
  • پیمایش مسیر (Path Traversal) — فرار از ساختار دایرکتوری تعیین شده
  • تزریق دستورات (Command Injection) — اجرای دستورات سیستم از طریق ورودی‌های تصفیه نشده
🔴 تزریق SQL
بر یکی از حیاتی‌ترین آسیب‌پذیری‌های وب مسلط شوید:
  • دور زدن احراز هویت — نفوذ به برنامه‌ها
  • تزریق SQL کور (Blind) — استخراج داده بدون خروجی قابل مشاهده
  • حملات مبتنی بر زمان — استفاده از تأخیر برای استنباط اطلاعات
  • تزریق مبتنی بر UNION — ترکیب پرس‌وجوها برای استخراج داده
  • پرس‌وجوهای پشته‌ای (Stacked Queries) — اجرای چندین دستور
🔵 بهره‌برداری پیشرفته
بردارهای حمله پیچیده را کاوش کنید:
  • تغییر نوع در PHP — بهره‌برداری از مقایسه‌های غیرسخت‌گیرانه
  • تزریق شی — استفاده از نقص‌های سریال‌زدایی
  • مسمومیت لاگ — تبدیل لاگ‌های سرور به اجرای کد
  • XML External Entity (XXE) — خواندن فایل‌ها از طریق پردازشگرهای XML
  • حملات نشست (Session) — پیش‌بینی و دستکاری شناسه‌های نشست
🟣 حملات آپلود فایل
خطرات قابلیت آپلود فایل را درک کنید:
  • دور زدن اعتبارسنجی سمت کلاینت — دور زدن بررسی‌های فرانت‌اند
  • دور زدن اعتبارسنجی سمت سرور — فرار از تأیید تصویر
  • بهره‌برداری از Magic Bytes — استفاده از هدر فایل‌ها برای پنهان کردن محموله‌ها

// این راهنما برای چه کسانی است؟

این راهنما برای طیف گسترده‌ای از مخاطبان طراحی شده است:

  • ▸ مبتدیان — تازه وارد امنیت وب هستید؟ از مرحله ۰ شروع کنید و مهارت‌های خود را گام به گام بسازید.
  • ▸ توسعه‌دهندگان — با درک نحوه کار آسیب‌پذیری‌ها، کدنویسی امن را بیاموزید.
  • ▸ علاقه‌مندان به امنیت — درک خود را از امنیت برنامه‌های وب عمیق‌تر کنید.
  • ▸ بازیکنان CTF — از این به عنوان یک مرجع و منبع یادگیری برای چالش‌های مشابه استفاده کنید.
  • ▸ دانشجویان — آموزش امنیت سایبری خود را با تمرین عملی تکمیل کنید.

هیچ تجربه قبلی در امنیت وب مورد نیاز نیست — فقط یک درک پایه از نحوه کار وب‌سایت‌ها و تمایل به یادگیری کافی است.
