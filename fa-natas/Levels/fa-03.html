۰۳ robots.txt — افشای مسیرهای پنهان ساده
// مفاهیم کلیدی
robots.txt افشای اطلاعات خزنده‌های وب امنیت از طریق پنهان‌کاری
// مرور کلی چالش

صفحه می‌گوید: "Not even Google will find it this time..." — یک اشاره مستقیم به فایل robots.txt که به خزنده‌های موتورهای جستجو دستور می‌دهد کدام مسیرها را ایندکس نکنند.

// robots.txt چیست؟
📄 robots.txt یک استاندارد برای راهنمایی خزنده‌های وب است. نکته مهم این است که این فایل به صورت عمومی قابل دسترسی است — یعنی هر مسیری که می‌خواهید از ایندکس شدن خارج کنید، به صراحت برای همه لیست شده است.
// تحلیل اولیه

اشاره "Not even Google will find it" یک اشاره واضح به robots.txt است. این فایل معمولاً در ریشه یک وب‌سایت قرار دارد و برای ارتباط با ربات‌های موتور جستجو استفاده می‌شود. اگرچه برای اهداف مشروع استفاده می‌شود (مانند جلوگیری از محتوای تکراری یا حذف مناطق مدیریتی از نتایج جستجو)، یک نقص عمده دارد:

  • فایل به صورت عمومی قابل دسترسی است
  • به طور صریح مسیرهایی را که مالک سایت می‌خواهد "پنهان" کند، لیست می‌کند
  • این امنیت از طریق پنهان‌کاری است — که اصلاً امنیت محسوب نمی‌شود
// مراحل حل
  1. فایل robots.txt را در ریشه وب‌سایت باز کنید:
http://natas3.natas.labs.overthewire.org/robots.txt
  1. محتوای فایل را مشاهده کنید:
# robots.txt for natas3 User-agent: * Disallow: /s3cr3t/
  1. به دایرکتوری ممنوع شده — /s3cr3t/ — بروید.
  2. لیست دایرکتوری فعال است — لیستی از فایل‌ها را مشاهده خواهید کرد.
  3. فایل users.txt را باز کنید تا رمز عبور را دریافت کنید.
http://natas3.natas.labs.overthewire.org/s3cr3t/users.txt
# username:password natas4:Z9tkRkWmpt9Qr7XrR5jWRkgOU901swEZ
// چرا کار می‌کند

فایل robots.txt برای خزنده‌های موتور جستجو طراحی شده است، نه برای امنیت. وقتی توسعه‌دهنده Disallow: /secret/ را اضافه می‌کند، در واقع به هر موتور جستجو (و هر مهاجمی) دقیقاً می‌گوید که محتوای پنهان را کجا پیدا کند. این یک مثال کلاسیک از امنیت از طریق پنهان‌کاری است — تکیه بر پنهان بودن چیزی که در واقع پنهان نیست.

// مشکل robots.txt
  • به صورت عمومی قابل دسترسی است
  • مسیرهایی که می‌خواهید پنهان کنید را تبلیغ می‌کند
  • مهاجمان به طور معمول این فایل را برای یافتن مسیرهای حساس بررسی می‌کنند
  • یک سیگنال برای مهاجمان است — "برای محتوای حساس اینجا را نگاه کنید"
// اقدامات دفاعی
  • هرگز به robots.txt برای امنیت تکیه نکنید — این یک دستورالعمل خزیدن است، نه یک مکانیزم کنترل دسترسی.
  • برای محافظت از دایرکتوری‌های حساس از احراز هویت و مجوز مناسب استفاده کنید.
  • در صورت امکان، فایل‌های حساس را خارج از ریشه وب ذخیره کنید.
  • اگر باید خزنده‌ها را مسدود کنید، از متا تگ‌ها یا هدرهای HTTP استفاده کنید.
💡 robots.txt برای SEO طراحی شده است، نه برای امنیت. استفاده از آن برای پنهان کردن مسیرهای حساس اثر معکوس دارد — آن مسیرها را به هر کسی که بررسی کند تبلیغ می‌کند.
// مشخصات ورود
natas3 sJIJNW6ucpu6HPZ1ZAchaDtwd7oGrD14
natas4 Z9tkRkWmpt9Qr7XrR5jWRkgOU901swEZ
// نکات کلیدی
🔍 امنیت ≠ پنهان‌کاری

پنهان کردن یک مسیر در robots.txt فقط توجه را به آن جلب می‌کند. امنیت واقعی نیازمند کنترل‌های دسترسی مناسب است، نه پنهان‌کاری.

🤖 robots.txt عمومی است

این فایل برای خزنده‌ها طراحی شده است، نه برای امنیت. همیشه عمومی است و هرگز نباید حاوی مسیرهای حساس باشد.
