۰۰ مشاهده کد منبع — کامنت پنهان ساده
// مفاهیم کلیدی
کامنت HTML مشاهده کد منبع افشای اطلاعات
// مرور کلی چالش

صفحه یک پیام خوش‌آمدگویی ساده بدون فرم یا ورودی نمایش می‌دهد. رمز عبور سطح بعدی مستقیماً درون یک کامنت HTML در کد منبع صفحه قرار گرفته است.

// تحلیل اولیه

وقتی یک صفحه وب بارگذاری می‌شود، مرورگر HTML را رندر می‌کند. با این حال، تمام کامنت‌های HTML (<!-- ... -->) توسط مرورگر نادیده گرفته می‌شوند اما همچنان به طور کامل در کد منبع قابل مشاهده هستند. این چالش نشان می‌دهد که چرا داده‌های حساس هرگز نباید در کد سمت کاربر قرار گیرند.

// مراحل حل
  1. به http://natas0.natas.labs.overthewire.org بروید و با مشخصات سطح وارد شوید.
  2. کلید Ctrl+U (یا Cmd+U در مک) را فشار دهید تا کد منبع باز شود.
  3. کامنت HTML حاوی رمز عبور را پیدا کنید.
<!-- the password for natas1 is gtVrDuiDfck83lPqWsLEZy5gyDz1clto -->
💡 کامنت‌های HTML در مرورگر نامرئی هستند اما در کد منبع به طور کامل قابل خواندن هستند. هرگز اطلاعات حساس را درون کامنت‌های HTML ذخیره نکنید.
// چرا کار می‌کند

HTML یک زبان سمت کاربر است. همه چیزهایی که به مرورگر ارسال می‌شوند — از جمله کامنت‌ها — از طریق شبکه منتقل می‌شوند و توسط کاربر قابل بررسی هستند. این یک آسیب‌پذیری در HTML نیست بلکه یک پیکربندی نادرست در نحوه مدیریت داده‌های حساس توسط برنامه است.

// اقدامات دفاعی
  • هرگز اعتبارنامه‌ها، کلیدهای API یا رمزهای عبور را در کامنت‌های HTML ذخیره نکنید.
  • همه منطق احراز هویت و مجوز باید در سمت سرور باشد.
  • برای تمام عملیات‌های حساس، اعتبارسنجی مناسب در سمت سرور پیاده‌سازی کنید.
// مشخصات ورود
natas0 natas0
natas1 gtVrDuiDfck83lPqWsLEZy5gyDz1clto
// نکات کلیدی
🔍 سمت کاربر ≠ امن

همه چیزهایی که به مرورگر ارسال می‌شوند توسط کاربر قابل خواندن هستند. هرگز به کد سمت کاربر برای امنیت اعتماد نکنید.

🛡️ دفاع در عمق

کنترل‌های امنیتی را در سمت سرور پیاده‌سازی کنید، نه فقط در سمت کاربر. کامنت‌ها برای توسعه‌دهندگان هستند، نه برای رازها.
