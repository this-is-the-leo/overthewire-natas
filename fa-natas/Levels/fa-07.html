۰۷ Path Traversal — عبور از دایرکتوری پیشرفته
// مفاهیم کلیدی
Path Traversal Directory Traversal افشای فایل اعتبارسنجی ورودی ../ (نقطه-نقطه-اسلش)
// مرور کلی چالش

این سطح یک صفحه با یک لینک "Home" نمایش می‌دهد که شامل یک پارامتر URL به نام page است. به نظر می‌رسد برنامه بر اساس این پارامتر فایل‌ها را می‌خواند و نمایش می‌دهد، که آن را در برابر Path Traversal آسیب‌پذیر می‌کند.

// تحلیل اولیه

ساختار URL یک پارامتر را نشان می‌دهد که برای بارگذاری محتوا استفاده می‌شود:

http://natas7.natas.labs.overthewire.org/index.php?page=home

به لینک‌های ناوبری توجه کنید:

home">Home about">About

این نشان می‌دهد که سرور کاری مشابه این انجام می‌دهد:

<?php include("pages/" . $_GET['page'] . ".html"); ?>
💡 Path Traversal به مهاجمان اجازه می‌دهد با استفاده از دنباله‌های ../ برای حرکت به بالا در درخت دایرکتوری، فایل‌های خارج از دایرکتوری مورد نظر را بخوانند.
// درک Path Traversal

Path Traversal (که به عنوان Directory Traversal یا حمله ../ نیز شناخته می‌شود) یک آسیب‌پذیری است که زمانی رخ می‌دهد که:

  • از ورودی کاربر برای ساخت مسیرهای فایل استفاده شود
  • هیچ اعتبارسنجی یا پالایشی انجام نشود
  • کاراکترهای خاص مانند ../ فیلتر نشوند
  • مهاجمان بتوانند خارج از دایرکتوری مورد نظر حرکت کنند

دنباله ../ در سیستم‌های فایل برای حرکت به یک سطح بالاتر استفاده می‌شود. با ترکیب چندین ../، مهاجم می‌تواند به هر فایلی در سیستم که سرور وب به آن دسترسی دارد برسد.

// مراحل حل
  1. پارامتر URL را مشاهده کنید: ?page=home.
  2. سعی کنید با استفاده از ../ فایل‌های خارج از دایرکتوری pages/ را بخوانید:
http://natas7.natas.labs.overthewire.org/index.php?page=../../../../etc/passwd
  1. برنامه فایل را شامل می‌کند و محتویات آن را نمایش می‌دهد.
  2. از آنجایی که رمز عبور در فایل /etc/natas_webpass/natas8 ذخیره شده است، سعی کنید به آن دسترسی پیدا کنید:
http://natas7.natas.labs.overthewire.org/index.php?page=../../../../etc/natas_webpass/natas8
  1. رمز عبور در صفحه نمایش داده می‌شود.

روش جایگزین — استفاده از curl:

curl -u natas7:7z3hEENjQtflzgnT29q7wAvMNfZdh0i9 \ "http://natas7.natas.labs.overthewire.org/index.php?page=../../../../etc/natas_webpass/natas8"
// چرا کار می‌کند

آسیب‌پذیری به دلایل زیر وجود دارد:

  • بدون اعتبارسنجی ورودی: پارامتر page پالایش نمی‌شود.
  • الحاق مسیر: برنامه مستقیماً ورودی کاربر را با یک مسیر پایه الحاق می‌کند.
  • بدون محدودیت: هیچ بررسی برای جلوگیری از خروج از دایرکتوری مورد نظر وجود ندارد.
  • دسترسی به سیستم فایل: سرور وب مجوزهای لازم برای خواندن فایل‌های هدف را دارد.
// اقدامات دفاعی
  • هرگز از ورودی کاربر مستقیماً در مسیرهای فایل استفاده نکنید.
  • از یک لیست سفید از مقادیر مجاز استفاده کنید.
  • اگر شامل کردن پویا ضروری است، تمام ورودی را اعتبارسنجی و پالایش کنید.
  • از basename() یا realpath() برای حل و اعتبارسنجی مسیر نهایی استفاده کنید.
  • سرور وب را با مجوزهای محدود اجرا کنید.
// مشخصات ورود
natas7 7z3hEENjQtflzgnT29q7wAvMNfZdh0i9
natas8 DBfUBfqQG69KvJvJ1iAbMoIpwSNQ9bWe
// نکات کلیدی
🔍 مراقب ../ باشید

ورودی کاربر حاوی دنباله‌های ../ می‌تواند خارج از دایرکتوری مورد نظر حرکت کند. همیشه اعتبارسنجی و پالایش کنید.

📋 لیست سفید بهترین راه است

استفاده از لیست سفید از مقادیر مجاز، ساده‌ترین و امن‌ترین رویکرد برای شامل کردن پویای فایل است.
