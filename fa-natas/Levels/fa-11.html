۱۱ رمزنگاری XOR — دور زدن رمزنگاری ضعیف پیشرفته
// مفاهیم کلیدی
رمزنگاری XOR رمزنگاری ضعیف دستکاری کوکی حمله متن ساده شناخته شده دور زدن رمزنگاری
// مرور کلی چالش

این سطح از یک کوکی برای ذخیره داده‌های کاربر استفاده می‌کند که با رمزنگاری XOR رمزگذاری شده است. کوکی حاوی رنگ پس‌زمینه ترجیحی کاربر و یک پرچم است که نشان می‌دهد کاربر وارد شده است یا خیر. با درک الگوریتم رمزنگاری XOR و استخراج کلید، می‌توانیم کوکی را تغییر دهیم تا به دسترسی دست یابیم.

// تحلیل اولیه

کد منبع را برای درک رمزنگاری بررسی کنید:

<?php $defaultdata = array( "showpassword" => "no", "bgcolor" => "#ffffff"); function xor_encrypt($in) { $key = '??????????????????????????????????'; $text = $in; $outText = ''; for($i=0; $i < strlen($text); $i++) { $outText .= $text[$i] ^ $key[$i % strlen($key)]; } return $outText; } function loadData($d) { $arr = json_decode(xor_encrypt(base64_decode($d)), true); if($arr) return $arr; die("invalid data"); } if(array_key_exists("data", $_COOKIE)) { $data = loadData($_COOKIE["data"]); } else { $data = $defaultdata; } if($data["showpassword"] == "yes") { print "Password: ..."; } ?>

کوکی حاوی داده‌های رمزگذاری شده با XOR است که با یک کلید مخفی رمزگذاری شده‌اند. هدف ایجاد یک کوکی جدید است که در آن showpassword برابر "yes" باشد.

💡 رمزنگاری XOR با کلید تکراری ضعیف است. اگر یک جفت متن ساده-متن رمز شده را بدانیم، می‌توانیم کلید را بازیابی کنیم و از آن برای رمزگذاری هر داده‌ای که می‌خواهیم استفاده کنیم.
// درک رمزنگاری XOR

XOR یک عملیات قابل بازگشت است:

  • متن ساده ^ کلید = متن رمز
  • متن رمز ^ کلید = متن ساده
  • متن ساده ^ متن رمز = کلید (اگر هر دو را بدانیم!)

از آنجایی که داده‌های پیش‌فرض (متن ساده) را می‌دانیم و کوکی (متن رمز) را داریم، می‌توانیم کلید را بازیابی کنیم.

// مراحل حل

مرحله ۱ — استخراج کوکی از مرورگر:

DevTools را باز کنید → Application → Cookies و مقدار کوکی data را کپی کنید.

مرحله ۲ — نوشتن یک اسکریپت PHP برای بازیابی کلید:

<?php $defaultdata = json_encode(array("showpassword" => "no", "bgcolor" => "#ffffff")); $cookie = "ClVLIh4ASCsCBE8lAxMacFMZV2hdVVotEhhUJQNVAmhSEV4sFxFeaAw="; $ciphertext = base64_decode($cookie); function xor_decrypt($ciphertext, $key) { $out = ''; for($i=0; $i < strlen($ciphertext); $i++) { $out .= $ciphertext[$i] ^ $key[$i % strlen($key)]; } return $out; } // بازیابی کلید $key = ''; for($i=0; $i < strlen($defaultdata); $i++) { $key .= $defaultdata[$i] ^ $ciphertext[$i]; } echo "Key: " . $key . "\n"; // خروجی: Key: qw8Jqw8Jqw8Jqw8Jqw8Jqw8Jqw8Jqw8Jqw8Jqw8Jqw8J // کلید "qw8J" است $key = "qw8J"; // ایجاد داده جدید با showpassword = "yes" $newdata = json_encode(array("showpassword" => "yes", "bgcolor" => "#ffffff")); $new_ciphertext = xor_decrypt($newdata, $key); $new_cookie = base64_encode($new_ciphertext); echo "New Cookie: " . $new_cookie . "\n"; ?>

مرحله ۳ — تنظیم کوکی جدید:

  1. از DevTools مرورگر برای تغییر کوکی data استفاده کنید.
  2. مقدار موجود را با مقدار جدید Base64 جایگزین کنید.
  3. صفحه را بازخوانی کنید.
// چرا کار می‌کند

رمزنگاری XOR با کلید تکراری به دلیل موارد زیر آسیب‌پذیر است:

  • حمله متن ساده شناخته شده: ساختار داده‌های پیش‌فرض را می‌دانیم.
  • کلید تکراری: کلید دوباره استفاده می‌شود، که به ما امکان بازیابی آن را می‌دهد.
  • بدون محافظت یکپارچگی: داده‌های رمزگذاری شده را می‌توان بدون شناسایی تغییر داد.
// اقدامات دفاعی
  • از رمزنگاری قوی مانند AES-256-GCM با مدیریت کلید مناسب استفاده کنید.
  • هرگز رمزنگاری خود را نسازید — از کتابخانه‌های تست‌شده استفاده کنید.
  • از یک کلید منحصر به فرد استفاده کنید و هرگز آن را بین داده‌های مختلف دوباره استفاده نکنید.
  • برای اطمینان از یکپارچگی، یک کد احراز هویت پیام (MAC) شامل کنید.
  • داده‌های حساس را به جای کوکی‌های سمت کاربر در سرور ذخیره کنید.
