26 PHP Object Injection — Deserialization Hard

// Key Concepts

PHP Object Injection Deserialization Serialized Data Remote Code Execution (RCE) __destruct()

// Challenge Overview

This level demonstrates PHP Object Injection (also known as PHP Deserialization). The application deserializes user-supplied data without proper validation, allowing an attacker to inject arbitrary objects that can lead to remote code execution.

// Initial Analysis

View the source code:

<?php class Logger { public $logFile; public $initMsg; public $exitMsg; function __construct() { $this->logFile = "img/log.txt"; $this->initMsg = "Log file initialized"; $this->exitMsg = "Log file closed"; } function __destruct() { file_put_contents($this->logFile, $this->initMsg . "\n" . $this->exitMsg); } } if (isset($_GET["data"])) { $data = base64_decode($_GET["data"]); $obj = unserialize($data); if (get_class($obj) == "Logger") { echo "Logger object created"; } } ?>

The application deserializes data from the data parameter and checks if it's a Logger object. The __destruct() method writes to a file using properties controlled by the attacker.

💡 PHP Object Injection occurs when user-supplied serialized data is deserialized without validation. Attackers can control the properties of objects, leading to unexpected behavior or remote code execution.

// Why It Works

The PHP object injection works because:

  • Unsafe deserialization: The application uses unserialize() on user input.
  • Controlled properties: The attacker can control the object's properties.
  • Magic method: The __destruct() method is automatically called, executing the attacker's code.
  • File write: The application writes to a file controlled by the attacker.

// PHP Magic Methods

Method When Called
__construct()When the object is created
__destruct()When the object is destroyed
__sleep()When serializing
__wakeup()When deserializing
__toString()When used as a string

// Defensive Measures

  • Never deserialize untrusted data.
  • Use JSON instead of serialize() and unserialize().
  • If deserialization is necessary, use a whitelist of allowed classes.
  • Use PHP 8.1+ with allowed_classes parameter in unserialize().
  • Sanitize all user input before using it in file operations.

// Secure Deserialization Example

<?php // Method 1: Allow only specific classes (PHP 8.1+) $obj = unserialize($data, ['allowed_classes' => ['AllowedClass']]); // Method 2: Validate the object after deserialization $obj = unserialize($data); if (get_class($obj) !== 'ExpectedClass') { die('Invalid class'); } // Method 3: Use JSON instead of serialization $data = json_decode($json_data, true); ?>

// Key Takeaways

📦 Deserialization Is Dangerous

Deserializing user-supplied data can lead to remote code execution. Never deserialize untrusted data without proper safeguards.

🛡️ Use JSON Instead

JSON is a safer alternative to PHP serialization. It doesn't support object instantiation or magic method execution.
