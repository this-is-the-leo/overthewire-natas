// Challenge Overview

This level has an advanced filter that attempts to block SQL injection by removing common SQL keywords. However, the filter can be bypassed using case sensitivity tricks and alternative comment styles.

// Initial Analysis

View the source code:

<?php $input = $_GET['username']; $filtered = preg_replace("/[;']/", "", $input); $query = "SELECT * FROM users WHERE username = '" . $filtered . "'"; $result = mysql_query($query); if (mysql_num_rows($result) > 0) { print "User exists"; } ?>

The filter removes ; and ' characters, but we can use other techniques to bypass it.

💡 Blacklist filters are often incomplete. Attackers can use case sensitivity (UNION vs union), URL encoding, or alternative comment syntax to bypass them.

// Why It Works

The SQL injection filter bypass works because:

  • Case sensitivity: The filter only checks for lowercase keywords.
  • Incomplete blacklist: Not all SQL keywords are blocked.
  • Comment alternatives: Multiple comment styles exist.
  • URL encoding: Spaces and special characters can be encoded.

// Common SQL Filter Bypasses

# Case sensitivity Union SeLeCt uNiOn sElEcT # Comment bypasses -- # MySQL, PostgreSQL, SQLite # # MySQL /**/ # MySQL, PostgreSQL, SQLite ;%00 # Null byte injection (older PHP versions) # Keyword bypasses or # OR || # OR and # AND && # AND

// Defensive Measures

  • Use prepared statements with parameterized queries.
  • Use a whitelist of allowed characters and keywords.
  • Case-insensitive filtering when using blacklists.
  • Input validation with a positive security model.
  • Use an ORM that automatically prevents SQL injection.

// Secure Code Example

<?php // Method 1: Prepared statements $stmt = $db->prepare("SELECT * FROM users WHERE username = ?"); $stmt->bind_param("s", $_GET['username']); $stmt->execute(); // Method 2: Whitelist $allowed_usernames = ['admin', 'user', 'guest']; $username = in_array($_GET['username'], $allowed_usernames) ? $_GET['username'] : ''; if ($username) { $stmt = $db->prepare("SELECT * FROM users WHERE username = ?"); $stmt->bind_param("s", $username); $stmt->execute(); } ?>

// Key Takeaways

🚫 Filters Can Be Bypassed

Blacklist filters are never perfect. Attackers can use case sensitivity, encoding, and alternative syntax to bypass them.

🛡️ Prepared Statements Are the Only Solution

Parameterized queries completely prevent SQL injection. No other defense is as effective or reliable.
