33 XXE — File Inclusion via External Entities Hard

// Key Concepts

XXE File Inclusion External Entity SSRF XML Parsing

// Challenge Overview

This level uses XXE to include files from the server. The application processes XML and displays the content of included files. We can use this to read the password file and other sensitive information.

// Initial Analysis

View the source code:

<?php if (isset($_POST['xml'])) { $xml = $_POST['xml']; $doc = new DOMDocument(); $doc->loadXML($xml, LIBXML_NOENT | LIBXML_DTDLOAD); echo $doc->textContent; } ?>

The application uses LIBXML_NOENT and LIBXML_DTDLOAD options, which enable entity substitution and DTD loading, making it vulnerable to XXE.

💡 LIBXML_NOENT enables entity substitution, which allows external entities to be resolved. This is a common XXE vulnerability.

// Why It Works

The XXE file inclusion works because:

  • External entities enabled: LIBXML_NOENT allows entity substitution.
  • DTD loading: LIBXML_DTDLOAD allows DTD loading.
  • File access: The parser has access to the filesystem.
  • PHP filters: PHP wrappers allow filtering the output.

// Defensive Measures

  • Disable external entity resolution in XML parsers.
  • Use libxml_disable_entity_loader(true) in PHP.
  • Don't use LIBXML_NOENT or LIBXML_DTDLOAD with untrusted input.
  • Use XMLReader instead of DOMDocument when possible.
  • Validate and sanitize all XML input.

// Secure XML Parsing Example

<?php if (isset($_POST['xml'])) { $xml = $_POST['xml']; // Disable external entities libxml_disable_entity_loader(true); // Parse without external DTDs $doc = new DOMDocument(); $doc->loadXML($xml); // Or use a schema to validate if ($doc->schemaValidate('schema.xsd')) { echo $doc->textContent; } } ?>

// Key Takeaways

📄 XXE Enables File Disclosure

XXE attacks can read any file on the server, leading to complete compromise of sensitive data.

🛡️ Disable External Entities

The most effective defense against XXE is to disable external entity resolution in XML parsers.
