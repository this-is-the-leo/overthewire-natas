23 PHP Type Juggling — Loose Comparison Bypass Hard

// Key Concepts

PHP Type Juggling Loose Comparison Authentication Bypass Type Coercion String-to-Integer Conversion

// Challenge Overview

This level uses PHP's loose comparison (==) to check a password. Due to PHP's type juggling, certain strings can be interpreted as integers, allowing us to bypass the comparison.

// Initial Analysis

View the source code:

<?php $passwd = "iloveyou"; // This is the actual password if(array_key_exists("passwd", $_GET)){ if(strstr($_GET["passwd"], "iloveyou")){ print "Password: ..."; } else { print "Wrong password!"; } } ?>

The application uses strstr() which checks if the string "iloveyou" is contained in the user input. However, there's a subtle issue with type juggling.

💡 PHP's loose comparison (==) and functions like strstr() can be tricked by type juggling. When comparing a string to an integer, PHP converts the string to a number.

// Understanding PHP Type Juggling

When PHP compares different types, it converts them to a common type:

  • "123" == 123 → True (string converted to integer)
  • "123abc" == 123 → True (string converted to integer)
  • "abc" == 0 → True (string converted to integer)
  • "0" == 0 → True (string converted to integer)

In this challenge, strstr() checks if one string contains another. But the function is used with a string and a variable that might be treated differently.

// Why It Works

The vulnerability exists because:

  • Password is hardcoded: The password is visible in the source code.
  • No hashing: The password is stored in plain text.
  • strstr() check: The function checks if the string is contained, which is a weak validation method.

// PHP Type Juggling Examples

// Loose comparison examples 0 == "0" // True 0 == "abc" // True 1 == "1abc" // True 0 == "0abc" // True "0abc" == 0 // True // strstr() examples strstr("hello world", "world") // Returns "world" (truthy) strstr("hello", "world") // Returns false

// Defensive Measures

  • Use strict comparison (===) instead of loose comparison (==).
  • Never hardcode passwords in source code.
  • Use password hashing (e.g., password_hash() and password_verify()).
  • Validate user input properly before using it in sensitive operations.
  • Use constant-time comparison for security-sensitive checks.

// Key Takeaways

⚖️ PHP Type Juggling Is Dangerous

Loose comparison (==) in PHP can lead to unexpected results. Always use strict comparison (===) for security-sensitive operations.

🔐 Never Hardcode Passwords

Passwords should never be stored in plain text in source code. Always use proper hashing and verification.
