24 PHP Type Juggling — Advanced Loose Comparison Hard

// Key Concepts

PHP Type Juggling Loose Comparison Authentication Bypass String-to-Integer Conversion Hash Comparison

// Challenge Overview

This level builds on Level 23 but with a more complex comparison. The application compares a user-supplied password with a hardcoded value using loose comparison (==). Due to PHP type juggling, we can bypass the check by providing a string that converts to the same integer value.

// Initial Analysis

View the source code:

<?php $passwd = "12345"; // This is the actual password if(array_key_exists("passwd", $_GET)){ if($_GET["passwd"] == $passwd){ print "Password: ..."; } else { print "Wrong password!"; } } ?>

The application uses loose comparison (==) to check the password. The password is "12345".

💡 In PHP, when comparing a string to an integer, the string is converted to a number. Strings like "12345abc" will convert to 12345, bypassing the comparison.

// Understanding the Vulnerability

PHP type juggling with loose comparison:

  • 12345 == "12345" → True (both are numbers)
  • 12345 == "12345abc" → True (string converts to 12345)
  • 12345 == "abc12345" → False (string starts with letters → converts to 0)
  • 12345 == "12345" → True

By using a string like "12345abc" or "12345xyz", we can bypass the password check.

// Why It Works

The type juggling vulnerability exists because:

  • Loose comparison: The application uses == instead of ===.
  • String-to-integer conversion: PHP converts strings to integers when comparing.
  • Leading characters matter: Only the leading numeric part of a string is converted.

// Defensive Measures

  • Use strict comparison (===) instead of loose comparison (==).
  • Type cast the values before comparison to ensure they are of the same type.
  • Use password hashing instead of plain text comparison.
  • Validate the input type before comparison.

// Secure Comparison Example

<?php $stored_password = "12345"; $user_input = $_GET['passwd']; // Method 1: Strict comparison if ($stored_password === $user_input) { echo "Password: ..."; } // Method 2: Type cast before comparison if ((string)$stored_password === (string)$user_input) { echo "Password: ..."; } // Method 3: Use password_hash and password_verify $hash = password_hash("12345", PASSWORD_DEFAULT); if (password_verify($user_input, $hash)) { echo "Password: ..."; } ?>

// Key Takeaways

⚖️ Strict Comparison Is Essential

Always use strict comparison (===) for security-sensitive operations. Loose comparison (==) can lead to type juggling vulnerabilities.

🔐 Hash Passwords, Don't Compare Plain Text

Use password_hash() and password_verify() for secure password storage and verification.
