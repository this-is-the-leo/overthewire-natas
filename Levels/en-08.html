// Challenge Overview

This level presents a form asking for a "secret" input. Viewing the page source reveals a PHP function that validates the input by applying a series of transformations. The secret is hidden in the source code — but not as plain text. It has been encoded using multiple steps.

// Initial Analysis

Inspect the page source to find the PHP code:

<?php $encodedSecret = "3d3d516343746d4d6d6c315669563362"; function encodeSecret($secret) { return bin2hex(strrev(base64_encode($secret))); } if(array_key_exists("submit", $_POST)) { if($encodedSecret == encodeSecret($_POST['secret'])) { print "Access granted. The password for natas9 is ..."; } else { print "Wrong secret"; } } ?>

The code shows the validation process:

  1. Take the user's input ($secret)
  2. base64_encode() — Encode to Base64
  3. strrev() — Reverse the string
  4. bin2hex() — Convert to hexadecimal
  5. Compare with $encodedSecret
💡 The secret is encoded in the source code, not stored as plain text. To find the original secret, we need to reverse the encoding process step by step.

// Understanding the Encoding

The encoding process uses three functions:

  • base64_encode() — Converts binary data to ASCII text using Base64 encoding
  • strrev() — Reverses a string (e.g., "hello" → "olleh")
  • bin2hex() — Converts a string to hexadecimal representation

To find the secret, we need to reverse these steps in the opposite order:

  1. hex2bin() — Convert hexadecimal back to binary string
  2. strrev() — Reverse the string
  3. base64_decode() — Decode from Base64

// Solution Walkthrough

Step 1 — Decode from Hex:

$encodedSecret = "3d3d516343746d4d6d6c315669563362"; $hex_decoded = hex2bin($encodedSecret); // Result: "==QcCtmMml1ViV3b"

Step 2 — Reverse the String:

$reversed = strrev($hex_decoded); // Result: "b3ViV1lmMtC cQ==" // Actually: "b3ViV1lmMtmC cQ==" → let's do this properly:

Let's write a PHP script to decode it:

<?php $encodedSecret = "3d3d516343746d4d6d6c315669563362"; // Step 1: Hex to binary $hex_decoded = hex2bin($encodedSecret); echo "Hex decoded: " . $hex_decoded . "\n"; // Output: ==QcCtmMml1ViV3b // Step 2: Reverse the string $reversed = strrev($hex_decoded); echo "Reversed: " . $reversed . "\n"; // Output: b3ViV1lmMtmCcQ== // Step 3: Base64 decode $secret = base64_decode($reversed); echo "Secret: " . $secret . "\n"; // Output: oubWYf2kBq ?>

Or using Python:

import base64 import binascii encoded_secret = "3d3d516343746d4d6d6c315669563362" # Step 1: Hex to bytes hex_decoded = bytes.fromhex(encoded_secret) print(hex_decoded) # b'==QcCtmMml1ViV3b' # Step 2: Reverse the string reversed_bytes = hex_decoded[::-1] print(reversed_bytes) # b'b3ViV1lmMtmCcQ==' # Step 3: Base64 decode secret = base64.b64decode(reversed_bytes) print(secret) # b'oubWYf2kBq'

Using command line (Linux/macOS):

# Step 1: Hex to ASCII echo "3d3d516343746d4d6d6c315669563362" | xxd -r -p # Output: ==QcCtmMml1ViV3b # Step 2: Reverse (using rev command) echo "3d3d516343746d4d6d6c315669563362" | xxd -r -p | rev # Output: b3ViV1lmMtmCcQ== # Step 3: Base64 decode echo "3d3d516343746d4d6d6c315669563362" | xxd -r -p | rev | base64 -d # Output: oubWYf2kBq

Enter oubWYf2kBq into the form on the page.

// Why It Works

The application stores the secret in encoded form, but the encoding is:

  • Not cryptographic: It's simple encoding, not encryption.
  • Reversible: All steps can be reversed with built-in functions.
  • Exposed in source code: The encoded value is visible to anyone who views the source.
  • No secret key: There's no key or salt required to decode.

// The Problem with Encoding vs. Encryption

  • Encoding (Base64, Hex) — Transforms data but provides no secrecy. It is easily reversible.
  • Hashing (SHA, MD5) — One-way function; cannot be reversed (but can be brute-forced).
  • Encryption (AES, RSA) — Uses a secret key; cannot be reversed without the key.
💡 Base64 encoding is not encryption! It's designed for data transmission, not security. Always use proper cryptographic hashing or encryption for sensitive data.

// Defensive Measures

  • Never store secrets in source code — use environment variables or configuration files outside the web root.
  • Use cryptographic hashing (e.g., bcrypt, Argon2) with salts for password storage.
  • Do not rely on encoding for security — Base64, Hex, and URL encoding are not secure.
  • Validate all user input — The server should validate the secret against a secure hash, not a simple encoded string.
  • Keep secrets out of version control — Use .env files or secret management tools.

// Secure Secret Storage Example

✅ Secure approach (hashing):

<?php // Store this hash in the database or config (not in code) $stored_hash = '$2y$10$92IXUNpkjO0rOQ5byMi.Ye4oKoEa3Ro9llC/.og/at2.uheWG/igi'; // When user submits a secret $user_input = $_POST['secret']; if (password_verify($user_input, $stored_hash)) { echo "Access granted!"; } else { echo "Access denied!"; } ?>

⚠️ Never do this: Storing encoded secrets in source code like $secret = "base64_string";

// Key Takeaways

🔐 Encoding ≠ Encryption

Base64, Hex, and URL encoding are not secure. They can be easily reversed and should not be used to protect secrets.

📝 Don't Hardcode Secrets

Never store passwords, API keys, or secrets directly in source code. Use environment variables or secure configuration files.
